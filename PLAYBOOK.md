# iSite PLAYBOOK（执行速查版）

## 使用顺序
1. 定义范围
2. 找楼识别
3. 整理证据
4. 白盒评分
5. 输出交付

## 一页判断
- 先全量，不预设每国数量
- 先证据标准化，再评分
- 高分不等于高证据，必须同时看 evidence_grade
- proxy 可以用，但必须标准化、降权、封顶
- 不在单物业备注里重复写通用说明

## 五维主评分
Total Score = 20 × (0.24×PV + 0.28×CM + 0.18×PU + 0.18×BR + 0.12×CX)

## 证据约束层
以下字段不直接加分，只用于封顶和分层：
- evidence_grade
- proxy_level
- hard_metric_completeness
- multi_source_check
- cap_reason

## 证据—评分映射
- 地位、规模、区位 → PV
- 规模、峰值、驻留、聚集、敏感度 → CM
- 用户结构、消费能力、商务/国际化 → PU
- 品牌、口碑、门户、示范效应 → BR
- 楼栋、园区、地下、连桥、热点复杂度 → CX

## PU / BR 边界
- PU 看“谁在用这个物业”
- BR 看“这个物业在外部认知中是什么地位”
- 同一条证据不得双重喂分
